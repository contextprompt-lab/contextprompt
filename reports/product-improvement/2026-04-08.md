## Problem

New users who sign up, connect repos, and land on the Recording page see
a blank space below the send-bot form. There's no indication of what will
happen when they paste a meeting link, no preview of the output, and no
visual encouragement. This "empty void" increases hesitation and drop-off
at the critical first-use moment.

Specific gap: `Recording.tsx` had no empty state for `meetings.length === 0`.

## Change

Added a first-meeting empty state to `website/app/src/pages/Recording.tsx`
(46 lines added, 0 removed).

The section renders **only** when:
```
!isActive && hasRepos && meetings.length === 0
```

It contains:
1. **Headline** — "Your first meeting is one paste away"
2. **Encouraging copy** — explains what happens after pasting a link
3. **3-icon flow** — Bot joins & records → Transcript generated → Tasks appear here
4. **Preview task card** — greyed-out (opacity 0.6), variant="outlined" card showing
   what a real completed task looks like (title, file chips, confidence badge)

Uses only MUI components and icons already imported (`SmartToyIcon`, `AccessTimeIcon`,
`TaskAltIcon`, `Card`, `CardContent`, `Chip`, `Stack`, `Box`, `Typography`, `Divider`).

## Rationale

Search Console shows the homepage converts at 33% CTR from position 2 (meta works).
The login page was improved last week with value props. The next bottleneck is
**first-use activation** — getting new users to record their first meeting.

Showing what the output looks like (before the user has committed to anything)
answers the implicit question "is this worth trying?" with a concrete visual answer.
