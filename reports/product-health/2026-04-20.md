## Dependency Audit (Monday)

**3 HIGH severity vulnerabilities** — all have fixes available.

| Package | Severity | Scope | CVE / Advisory | Fix |
|---------|----------|-------|----------------|-----|
| `path-to-regexp@8.0.0–8.3.0` | **HIGH** | **Production** (Express → router) | GHSA-j3q9-mxjg-w52f (ReDoS sequential optional groups, CVSS 7.5) + GHSA-27v5-c462-wpq7 (ReDoS multiple wildcards, CVSS 5.9) | Upgrade to `>=8.4.0` |
| `picomatch@4.0.0–4.0.3` | **HIGH** | Dev-only (vitest/vite) | GHSA-3v7f-55p6-f55p (method injection) + GHSA-c2c7-rcm5-vvqj (ReDoS extglob, CVSS 7.5) | Upgrade to `>=4.0.4` |
| `vite@<latest` | **HIGH** | Dev-only | Multiple CVEs (server bypass, path traversal) | Upgrade vite |

**`path-to-regexp` is a production vulnerability** affecting the Express routing layer. Exposure: any publicly routable URL with specially crafted path segments could cause CPU spin. Recommended fix — add to `package.json`:
```json
"overrides": {
  "path-to-regexp": "^8.4.0"
}
```
Then run `npm install`. This requires human action (agents may not add/modify npm dependencies per guardrails).

**Day 8 since first discovery** — this vulnerability has been flagged every day since 2026-04-12. Escalating urgency.
