Questions from developers

The things developers ask before trying contextprompt.

Does contextprompt work with Claude Code?

Yes. The task output format is designed to be pasted directly into Claude Code (or any AI coding assistant). Each task includes a description, affected files with paths, ordered implementation steps, and confidence level — exactly the structured context Claude Code needs to start implementing without follow-up questions.

How does it know which files in my repo are relevant?

contextprompt indexes your repository: it walks the file tree, respects your .gitignore, and extracts exported function names, type definitions, and interfaces using TypeScript's AST parser. When a task is extracted from a transcript, it uses keyword matching and semantic similarity to find the most relevant files — the same process a developer uses mentally when estimating a task.

Do I need to install anything?

No. contextprompt is entirely web-based. You connect your repos via the dashboard (or use the File System Access API to share a local repo without uploading files to our servers), and the bot joins meetings via a URL. There's no browser extension, no desktop app, and no IDE plugin required.

Which meeting platforms does it support?

contextprompt supports Zoom, Google Meet, Slack Huddles, and Microsoft Teams. Paste any meeting URL into the dashboard and the bot joins automatically as a participant. It works with any meeting where you can share a link — no host permissions or meeting admin access required.

Is it safe to connect my private repositories?

Yes. contextprompt uses the File System Access API so you can share your local repository structure with the browser without files being uploaded to our servers. The repo map (file paths, function names, type signatures) is sent for analysis but your actual source code stays local. You can also use a public GitHub repo URL directly.
