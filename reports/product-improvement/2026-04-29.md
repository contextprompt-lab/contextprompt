## Problem

MeetingDetail is the product's highest-value page — the moment after a meeting where users see their conversation turned into actionable coding tasks with specific file paths and implementation steps. This is the "aha moment."

But the page had no way to share all results at once. Users could copy individual tasks one-by-one (via the TaskCard copy icon), but had no way to export the full meeting output. And there was no conversion prompt at this peak-engagement moment.

The practical impact:
- Users who want to paste their tasks into Slack, Notion, or a GitHub PR description have to copy each task individually — friction that discourages sharing
- Teammates never see contextprompt output → no viral loop
- Free users who just got value from the product have no upgrade prompt at the right moment

## Change

Added to `website/app/src/pages/MeetingDetail.tsx`:

**1. "Copy all tasks" button** (next to the Tasks heading)
- Builds a formatted markdown document: meeting date → decisions list → all tasks (via existing `buildAgentBlock` helper in TaskCard)
- Copies to clipboard via `navigator.clipboard.writeText()`
- Transitions to "Copied! ✓" (green, check icon) for 2 seconds, then resets
- Hidden when there are 0 tasks

**2. Inline upgrade nudge** (below the task list, free users only)
- One line of caption text: "Need more than 1 hour of recording per month? Upgrade to Pro for 15 hours/month →"
- The link calls `createCheckoutSession()` → Stripe checkout (same flow as sidebar chip)
- Invisible to Pro users; invisible when tasks = 0

## Why this page, why now

The MeetingDetail page is where users decide whether the product is worth paying for. The extracted tasks — with file paths, steps, and confidence levels — are the proof of value. At this moment, the user:
1. Has already used the product (they sent a bot to a meeting)
2. Has seen the output (they're looking at tasks)
3. Is deciding what to do next (copy tasks, share with team, etc.)

This is the highest-intent moment in the product loop to prompt an upgrade.
